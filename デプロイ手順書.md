## 🪜 作業順序

### Step 1：Supabase で SQL 実行（RLSポリシー追加）

1. Supabase Dashboard → 左サイドバー **「SQL Editor」** をクリック
2. **「New query」** ボタン
3. `clients_self_update_policy.sql` の中身を**全コピー**して貼り付け
4. 右下 **「Run」** ボタン（または Cmd+Enter）
5. 「Success. No rows returned」と出れば成功

**確認方法：** SQL Editor に以下を貼って実行：
```sql
SELECT policyname FROM pg_policies WHERE tablename = 'clients';
```
→ `clients_self_update` が一覧に出ていればOK

---

### Step 2：Edge Function を v2 に更新

⚠️ 既存のEdge Functionに `update_email` アクションを追加するため、コードを上書きする必要があります。

1. Supabase Dashboard → 左サイドバー **「Edge Functions」**
2. `admin-user-manager` をクリック
3. 上部タブの **「Code」** をクリック
4. エディタの中身を**全削除**
5. `edge_function/supabase/functions/admin-user-manager/index.ts` の中身を**全コピー**して貼り付け
6. 右上の **「Deploy」** ボタンをクリック
7. デプロイ成功（10〜30秒）

---

### Step 3：フロント本番反映（GitHub）

⚠️ いつものGitHubアップロード手順です。

1. GitHubリポジトリ `mukaemura/saiyo-core` を開く
2. リポジトリ直下の `index.html` を `patch/index.html` で上書きアップロード
3. リポジトリの `js/` ディレクトリ → `app.js` を `patch/js/app.js` で上書きアップロード
4. GitHub Pages の反映待ち（1〜2分）
5. ブラウザで `https://app.link-core.co.jp/` を **強制リロード**（Cmd+Shift+R）

**確認方法：** DevToolsのNetworkタブで以下を確認：
- `app.js?v=20260510a` がロードされている（旧 `v=20260508i` ではない）
- ファイルサイズが約 **483KB** になっている

## 🚨 異常系のトラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| 「Authユーザーと紐付いていません」エラー | 古いクライアントで `auth_user_id` がNULLのまま | Supabase Authentication → Users で手動作成して、clientsテーブルで `auth_user_id` を手動セット |
| 「Forbidden: admin only」 | adminアカウントでない | adminアカウントでログインし直す |
| `update_email` で「Auth更新失敗」 | Edge Functionが古いまま（v1のまま） | Step 2 でEdge Functionをv2に更新したか再確認 |
| クライアント側パスワード変更後、admin画面のパスワード列が古いまま | RLSポリシー未追加 | Step 1 のSQLを実行したか再確認 |
