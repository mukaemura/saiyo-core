# 採用コア（軽量化版）

## 2種類のファイルがあります

### 1. 分離版（本番デプロイ用）— `saiyo-core/` フォルダ

```
saiyo-core/
├── index.html          # 画面構造のみ（66KB）
├── css/style.css       # スタイル定義（10KB）
├── js/app.js           # アプリケーションロジック（200KB）
└── assets/
    ├── logo.png             # メインロゴ（11KB / 9箇所で再利用）
    ├── character.png        # メインキャラクター（28KB / 6箇所で再利用）
    ├── character-small.png  # 空表示用キャラ（10KB）
    └── header-icon.png      # ヘッダーロゴ（5KB）
```

**合計サイズ: 約 322KB**（元: 650KB / 削減率 50%）

このフォルダを丸ごと Web サーバーや Vercel / Netlify / GitHub Pages 等にアップロードしてください。

ローカル動作確認:
```bash
cd saiyo-core
python3 -m http.server 8000
# → http://localhost:8000 にアクセス
```

⚠️ `file:///` で直接開くと CORS 制約で CSS/JS が読み込めません。HTTPサーバー経由で開いてください。

### 2. 単独版（プレビュー / メール添付 / 手軽な確認用）— `saiyo-core-standalone.html`

CSSとJSと画像をすべて1ファイルに統合した版（約630KB）。
- ✅ ダブルクリックで開ける（HTTPサーバー不要）
- ✅ プレビューでも画像が表示される
- ❌ 本番運用には非推奨（キャッシュが効かないため毎回フル読込）

## どちらを使うべきか

| 用途 | 推奨 |
|---|---|
| プレビューで動作確認 | 単独版 |
| 本番Webサーバーにデプロイ | 分離版 |
| クライアントへの動作デモ | 単独版 |
| GitHub等で管理・編集 | 分離版（編集しやすい）|

## 軽量化の主な変更点（分離版）

1. **Base64画像 → 外部ファイル化** — 画像が18箇所に Base64 で埋め込まれていた状態（実質4種類が重複）を `assets/` に分離。ブラウザがキャッシュするため、9回参照されているロゴも1回しかダウンロードされません
2. **CSS分離** → `css/style.css`
3. **JavaScript分離** → `js/app.js`
4. **未使用CSSクラスを削除**（13個 / 約1.5KB削減）
